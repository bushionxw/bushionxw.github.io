1.2strings

strings命令很简单， 看起来好像没什么， 但实际有很多用途。 下面， 我来举一个例子。 在大型的软件开发中， 假设有100个.c/.cpp文件， 这个.cpp文件最终生成10个.so库， 那么怎样才能快速知道某个.c/.cpp文件编译到那个.so库中去了呢？ 当然， 你可能要说， 看makefile不就知道了。 对， 看makefile肯定可以， 但如下方法更好， 直接用命令：

​ strings -f “*.so” | grep “xxxxxx”

查看依赖库

​ strings -a *.so | grep GCC

​ strings -a *.so | grep GLIB

strings命令在对象文件或二进制文件中查找可打印的字符串。字符串是4个或更多可打印字符的任意序列，以换行符或空字符结束。 strings命令对识别随机对象文件很有用。

2.gdb调试

2.1调试流程

(1).gdb programname ：进入调试

(2).break main 或者 break 行号 ：设置断点

(3).run ：开始执行

(4).step或者s ： 运行一行代码

​ 会进入调用函数内部

(5).Enter ：进入调用函数

(6).print 表达式 ：打印

​ 如： print input

(7).displaty 表达式 ：显示监视的数据

​ 如： display input

(8).next或者n :下一步

​ 不进入调用函数内部

(9).quit ：推出gdb调试

(10). finish :强制退出
